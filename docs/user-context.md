# User Context Template

> Status: review-needed | Owner: user | Last verified: 2026-07-23

Keep this public file generic. In a private fork or local installation, replace it with stable context that materially improves decisions. Do not store secrets, credentials, private addresses, health records, or information you would not want committed.

## Operational Profile

- Preferred name: replace locally.
- Role and working mode: replace locally.
- Location or timezone, only when operationally useful: replace locally.
- Time, accessibility, or collaboration constraints: replace locally.
- Primary assistant workflow and preferred level of autonomy: replace locally.

Record only facts the assistant would otherwise need to ask repeatedly.

## Brand and Product Architecture

Define the hierarchy explicitly:

1. Who is the public individual or team?
2. What organization or brand sits at the center?
3. Which products are flagship products?
4. Which tools are components of a larger product?
5. Which projects are secondary, experimental, archived, or stopped?

Do not let every project appear equally strategic. State the relationship between the brand, products, services, and public proof.

## Canonical Positioning

Write one sentence in each public language:

> [Name or team] creates [specific category] for [specific audience]. Through [organization], it builds [core value], with [flagship products] as the main proof.

Also record:

- Primary public label.
- Labels to avoid.
- Audience and user problem.
- Differentiating thesis.
- Evidence that supports the claim.

## Product and Distribution

- Current focus and measurable objective.
- Distribution channels that matter.
- Product claims and the artifacts that prove them.
- Monetization constraints.
- Explicitly rejected strategies.

## Career or Client Fit

- Types of work that fit.
- Types of work that do not fit.
- Target teams, companies, or collaborators.
- Location, language, availability, and employment constraints.
- Strategic tradeoffs the assistant should challenge.

## Writing Voice

Record concrete rules derived from corrections:

- Preferred sentence rhythm and level of formality.
- Openers, closers, and rhetorical patterns to avoid.
- Whether proofreading means typo-only or permits rewriting.
- Vocabulary that sounds natural or artificial.

Delete any section that does not improve future decisions. This file is context, not a biography.
